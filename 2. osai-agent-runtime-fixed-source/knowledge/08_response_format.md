# Response Format

> File guide:
> - Purpose: Defines preferred answer structure for operator-facing responses.
> - Where this fits in OSAI: Guides reasoning/AI output toward status, evidence, risk, and next checks.
> - Topics to know: Markdown structure, OSAI architecture, Docker services, Cognee memory, and llama.cpp/Qwen inference.
> - Operational note: Keep format concise so answers remain usable during incidents.



When answering the operator:

1. Start with the current status.
2. Mention the evidence.
3. Explain the likely cause.
4. Give safe next checks.
5. Only then suggest action.
6. Mark risky actions clearly.
